# Agent Onboarding: Day 1
### How to run the Austin Deal Desk in 5 minutes.

Welcome to the team. We don't use a complex CRM. We use **The Desk**. 
Follow these four steps to handle your first client request by lunch.

---

## 1. The Manila Envelope Rule
If a client texts, emails, or calls with a request, it doesn't exist until it’s in an **Envelope**.
1. Go to `MANILA_ENVELOPE.md`.
2. Copy the template content.
3. Create a new file in the root directory named `AUSTIN-YYYYMMDD-CLIENTNAME.md`.
4. Paste the template and fill in the "Intake" section.

## 2. Station-to-Station Movement
We don't "assign tasks." We move files.
- To start a deal: Drop the file in `00_orchestrator`.
- To check compliance: Move it to `01_lead_qualifier`.
- To research the house: Move it to `02_property_research`.
*Always leave a 1-sentence note in the file's Desk Trail before you move it.*

## 3. Handling the Slips
The AI specialists at each station will apply a "Slip" to the top of your file. 
- **BLUE SLIP:** Stop. This is a compliance or verification block. Do not send drafts to clients.
- **RED FLAG:** Warning. There is a physical or financial risk (e.g., foundation issues) you must disclose.
- **GREEN SLIP:** Clear. You are authorized to move to the next station.

## 4. Your First "Hard Stop"
If you try to schedule a showing but the file has no Buyer-Representation Agreement, the **01_Lead_Qualifier** will hit you with a **BLUE Slip**. 
*Action:* Do not call the showing service. Send the file to Diana. Once she clears it, the slip turns Green, and you can proceed to Station 03 (Comms) to book the tour.
