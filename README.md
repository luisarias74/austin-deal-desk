# The Austin Deal Desk
### The One-Week Operational Turnaround for Boutique Real Estate Teams.

**"Diana doesn’t need another app. She needs a desk system her team will actually use."**

The Austin Deal Desk is a folder-native AI operating system. No new logins. No CRM bloat. No "vibe coding." Just a specialized desk architecture designed to help protect the team from avoidable compliance and deadline mistakes while they are in the field.

---

## The Core Promise

1. **Onboard in 24 hours:** A new agent can learn the system by reading one file: `AGENT_ONBOARDING_DAY_1.md`.
2. **Zero-guessing compliance:** The BLUE Slip Hard Stop prevents a showing workflow from advancing until buyer-representation status is verified.
3. **The Desk Trail:** A permanent, reviewable record of every AI-assisted decision, property risk, handoff, and draft deadline calculation.

---

## The Flow: The Manila Envelope Doctrine

We do not move random tasks.
We move **Manila Envelopes**.
Every case travels through five specialized desk stations so context is not lost between intake, compliance, property research, communication, and contract tracking.

```text
[ INTAKE ] -> [ COMPLIANCE ] -> [ RESEARCH ] -> [ DRAFTING ] -> [ TRACKING ]
  (00)           (01)             (02)             (03)           (04)
Orchestrator   Lead Qual       Prop Research    Comms Desk     TC Desk
```

---

## Run Any Request In Five Steps

1. Open a Manila Envelope from `MANILA_ENVELOPE.md`.
2. Start new or unclear work in `00_orchestrator/`.
3. Apply the correct slip: RED, YELLOW, BLUE, or GREEN.
4. Move the envelope to one station only.
5. Update the Desk Trail before moving on.

---

## The Desk Stations

| Station | Owns | Never Owns |
|---|---|---|
| `00_orchestrator/` | Opens envelopes, checks completeness, routes work. | Specialist work, legal advice, deadline math. |
| `01_lead_qualifier/` | Intake, client readiness, buyer-representation/showing flags. | Property research, legal conclusions, final compliance approval. |
| `02_property_research/` | Property risk flags, Austin-specific research gaps, verification needs. | Engineering, zoning, floodplain, or legal conclusions. |
| `03_client_communication/` | Drafts client, listing-agent, co-op-agent, vendor, and internal messages. | Sending messages, negotiation decisions, legal advice. |
| `04_transaction_coordinator/` | Effective Date, option period, earnest money, appraisal, financing, deadline tracking. | Legal interpretation, repair strategy, client persuasion. |

## The Moat: The BLUE Slip Protocol

Unlike generic AI assistants, The Austin Deal Desk is built to refuse unsafe movement.

If the buyer-representation flag is missing:

1. The system blocks the showing workflow.
2. The AI applies a **BLUE Slip**: verification required.
3. The case is routed to Diana, broker, or the appropriate licensed professional for manual verification.

**No cleared buyer-representation flag = no showing workflow.**

**Pre-showing intake/research only — showing not cleared.**

## Required Manila Envelope Fields

Every request should capture or mark these fields as unknown:

- Case ID
- Client Name
- Client Type
- Request Type
- Property Address
- Current Station
- Current Stage
- Known Facts
- Missing Facts
- Risk Flags
- Compliance Flags
- Deadline Flags
- Next Requested Action
- Human Approval Requirement
- Desk Trail

Unknown facts stay marked as **Unknown**. Do not guess.

---

## Human Review Gates

Human review is required before:

- any client-facing message is sent
- any listing-agent or co-op-agent message is sent
- any pricing, offer, repair, or negotiation strategy is shared
- any legal or compliance-sensitive language is used
- any deadline-sensitive action is taken
- any RED or BLUE slip is cleared

The AI may draft, organize, summarize, and flag risk. It may not approve, send, commit, or decide on behalf of Diana’s team.

---

## Deadline Rules

**Effective Date = Day 0.**

**The next calendar day = Day 1.**

If Effective Date is missing: **YELLOW slip — Effective Date missing. Deadline calculation blocked.**

Any deadline table must say: **Draft deadline calculation — verify before relying.**

---

## Definitions Of Done

- **Orchestrator** is done when the request is classified, missing information is named, one destination folder is selected, and review/escalation is flagged.
- **Lead Qualification** is done when client readiness and buyer-representation/showing status are clear, blocked, or marked unknown.
- **Property Research** is done when verified facts are separated from unverified claims and Austin/Texas verification needs are marked.
- **Client Communication** is done when a draft exists, the audience is clear, the reviewer is named, and the draft is labeled: `Draft only — human approval required before sending.`
- **Transaction Coordination** is done when Effective Date status, deadline sources, responsible person, and communication approval/sent status are tracked.
