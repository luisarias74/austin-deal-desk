# 03 Client Communication Rules

## Rule 1: Draft Only

Every output is a draft.

The AI never sends.

Use this label:

```md
Draft only — human approval required before sending.
```

---

## Rule 2: Identify The Audience

Before drafting, identify whether the message is for:

- client
- listing agent
- co-op agent
- buyer agent
- seller agent
- inspector
- lender
- title company
- vendor
- internal team

---

## Rule 3: Do Not Make Decisions

This station must not decide:

- what Diana should offer
- whether to terminate
- whether to ask for repairs
- whether to negotiate credits
- whether a deadline is final
- whether a form is compliant
- whether a legal right exists

It may draft neutral language for Diana’s approval.

---

## Rule 4: Use Text-First Drafts When Helpful

For mobile use, include:

```md
## Text-First Draft

Draft only — human approval required before sending.

...
```

Keep it short.

---

## Rule 5: Outside-Agent Communication Requires Care

Messages to listing agents or co-op agents can affect negotiation and deal posture.

Apply RED or BLUE slips when the message involves:

- inspection defect
- repair request
- deadline pressure
- compliance issue
- contract term
- buyer/seller strategy
- potential dispute

---

## Rule 6: Carry The Context Forward

For return-trip handoffs, include:

- why the message is needed
- supporting facts
- what is verified
- what is not verified
- risk flags
- approval needed
- where the envelope should return

---

## Rule 7: Same-Day Acknowledgment

A fast reply may acknowledge a same-day buyer request and ask for missing items (lender letter, written representation, property address/MLS#).

It must not confirm a showing until buyer-representation/compliance and scheduling are cleared.

A same-day acknowledgment draft is still a draft. Diana or the assigned human must approve before it is sent.

---

## Rule 8: First-Touch Acknowledgment Ownership

`03_client_communication` may draft the first-touch acknowledgment and mark `First-Touch Acknowledgment Sent` as:

```md
Drafted / Awaiting Human Approval
```

It must not mark the acknowledgment as `Sent by Human` unless the assigned human confirms it was sent. When the human confirms, update the field and set `First-Touch Sent Timestamp` to the actual send time.

---

## Rule 9: Update Desk Trail

Example:

```md
[03-COMMS] 2026-05-12 — Drafted client text and email regarding inspection/foundation concern. Draft only. Diana approval required before sending. Returning envelope to 04_transaction_coordinator for communication tracking.
```
